# Architecture Decision Records (ADR)

Use ADRs for decisions that affect:

- package boundaries,
- public API contracts,
- CI/release/security policy,
- long-term operational model.

## Lifecycle

- `proposed` -> `accepted` -> `superseded` (if replaced).
- Never delete ADRs; append superseding links.

## Naming

- `NNNN-short-title.md` (e.g. `0001-monorepo-governance-baseline.md`)

## Template

- Start from [`0000-template.md`](./0000-template.md).
