# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any Affino project, **please do not open a public GitHub issue**.

Instead, report it privately by email:

ðŸ“§ **support@affino.dev**

Please include as much detail as possible to help us understand and reproduce the issue:
- A description of the vulnerability
- Steps to reproduce
- Affected package(s) and version(s)
- Any relevant logs, screenshots, or proof-of-concept code

We aim to acknowledge valid reports within **72 hours**.

---

## Supported Versions

Security fixes are applied to the **latest released versions** of the following packages:

- `@affino/menu-core`
- `@affino/menu-vue`
- `@affino/menu-react`
- `@affino/selection-core`
- `@affino/virtualization-core`

Older versions may not receive security updates. Users are encouraged to upgrade to the latest version whenever possible.

---

## Responsible Disclosure

We kindly ask that you:
- Give us reasonable time to investigate and fix the issue before any public disclosure
- Avoid exploiting the vulnerability beyond what is necessary to demonstrate it
- Avoid accessing or modifying data that does not belong to you

We are committed to working with security researchers in good faith.

---

## Safe Harbor

We consider security research conducted under this policy to be:
- Authorized
- In good faith
- In compliance with applicable laws

We will not pursue legal action against researchers who follow this policy and report vulnerabilities responsibly.

---

## Acknowledgements

We appreciate the efforts of the security community and are happy to acknowledge valid reports where appropriate.

Thank you for helping keep Affino secure.
